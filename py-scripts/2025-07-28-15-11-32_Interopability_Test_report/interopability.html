Objective

The Candela Interoperability test is designed to measure an Access Point’s client performance when handling different amounts of Real clients like android, Linux, windows, and IOS. The test allows the user to increase the number of clients in user-defined steps for each test iteration and measure the per-client throughput for each trial. Along with throughput other measurements made are client connection times, Station 4-Way Handshake time, DHCP times, and more. The expected behavior is for the AP to be able to handle several stations (within the limitations of the AP specs) and make sure all Clients get a fair amount of airtime both upstream and downstream. An AP that scales well will not show a significant overall throughput decrease as more Real clients are added.
